Browser Automation
üîó
CDP (Chrome DevTools Protocol) is your friend
Connect to an existing browser session instead of launching new ones. Preserves auth, cookies, and state.
üîÑ
Disconnect to bypass bot detection
Some CAPTCHAs (like Turnstile) won't resolve while CDP is connected. Disconnect, wait, reconnect.
üì∏
Save snapshots for debugging
Don't re-run automation to debug. Save HTML/screenshots locally and iterate on parsing logic.
üé≠
ariaSnapshot() over innerHTML
Accessibility tree is cleaner than raw HTML. Great for extracting structured data from complex pages.
# Playwright + CDP connection
const browser = await chromium.connectOverCDP('http://localhost:9222');
const page = browser.contexts()[0].pages()[0];

# Get accessibility tree (cleaner than HTML)
const snapshot = await page.locator(':root').ariaSnapshot();

API Integrations
üîê
Store keys in .env, never in code
One .env file, loaded automatically. Never commit secrets to git.
üîÅ
Retry with backoff
APIs fail. Wrap calls with retry logic: 1s ‚Üí 2s ‚Üí 4s delays between attempts.
üíæ
Cache aggressively
If data doesn't change often, cache it. We use 7-day TTL for external API responses.
‚è±Ô∏è
Always set timeouts
30s is a good default. Hanging requests block everything downstream.

Hard Lessons
üö´
Parallel browser sessions don't work
They share one profile = conflicts. Run browser automation sequentially.
üîÑ
Some sites just block automation
Cloudflare, aggressive rate limits, fingerprinting. Know when to give up and find another way.
üìù
Log everything during development
When automation fails at 3am, you'll want to know what happened. Verbose logs save hours.
